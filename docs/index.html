API 中转站 官方文档

基于 OpenAI 兼容协议的 API 代理服务，支持 GPT-5 系列、Codex、DeepSeek 等模型。 一行命令即可接入，零门槛使用顶尖 AI 模型。

概览

本中转站对上游多路 AI 模型进行统一代理，提供 OpenAI 兼容的 API 端点。 你只需一个 API Key，即可访问 ChatGPT、Codex、DeepSeek 等多个模型的文本对话与图片生成能力。

1

获取 API Key

在中转站管理后台创建令牌，获得 sk-xxxx 格式的密钥。

2

配置客户端

将 Base URL 和 API Key 填入 Codex、CCSwitch 或任意 OpenAI 兼容客户端。

3

开始调用

选择模型，发送对话请求，即可获得 AI 回复。

安装 Codex

Codex 是 OpenAI 推出的 AI 编程助手 CLI 工具。以下是安装步骤：

macOS / Linux

# 通过 Homebrew 安装
brew install openai/tap/codex

# 或通过 npm 全局安装
npm install -g @openai/codex

# 验证安装
codex --version

Windows

# 通过 winget 安装
winget install OpenAI.Codex

# 或通过 npm 全局安装
npm install -g @openai/codex

# 验证安装
codex --version

配置中转站

安装完成后，配置 Codex 指向中转站：

codex config set base_url https://yeshijie.top/v1
codex config set api_key sk-你的API密钥
提示：Base URL 末尾的 /v1 不能遗漏，否则 API 路径会拼接错误。

充值额度

重要：创建后会短暂显示完整 Key（sk-xxxx...），请立即保存。关闭后无法再次查看完整 Key。

导入 CCSwitch

CCSwitch 是 Claude / Codex / Gemini 的多账号快速切换工具，内置 CLIProxyAPI 集成。

方法一：从令牌管理一键导入（推荐）

在"令牌管理"列表中找到你的 Key → 点击右侧 ▼（倒三角） → 选择 "导入到 CCSwitch"。

▲ 点击倒三角 → 导入到 CCSwitch

方法二：手动配置 CCSwitch

# 在 CCSwitch 中添加渠道
渠道类型: OpenAI 兼容
Base URL: https://你的域名/v1
API Key:  sk-你的密钥

测试连接

▲ CCSwitch 渠道管理面板

找到导入的渠道，点击 "测试连接"：

结果含义
✅ 绿色成功API 连通，可正常使用
❌ 红色失败检查网络、Base URL 和 Key

也可用命令行测试：

常见错误码

HTTP 状态错误信息解决方案
401Invalid API Key检查 Key 是否正确、是否过期
403模型无权限检查令牌的模型限制和分组
429请求频率过高降低并发，稍后重试
500无可用渠道该模型在当前分组中没有可用的上游渠道
502上游服务不可用上游模型服务暂时故障

常见问题

忘记 API Key 怎么办？

Key 创建后仅展示一次。如丢失，请在令牌管理页 删除旧 Key 重新创建。

为什么模型列表是空的？

检查分组设置，确认渠道中已添加对应模型，且渠道状态为启用。

请求超时怎么办？

检查中转站服务器状态，长文本生成可能需要较长时间。流式模式建议设置 stream: true。

Key 泄漏了怎么办？

立即在令牌管理页 删除该 Key，然后创建新 Key。

支持哪些客户端？

任何支持 OpenAI 兼容 API 的客户端均可使用：CCSwitch、Codex、Cherry Studio、NextChat、LobeChat、OpenCat 等。
